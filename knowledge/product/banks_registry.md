# Реестр банков РФ — вселенная покрытия FINPILOT

> **Назначение.** Ответ на вопрос «сколько банков FINPILOT в пределе должен уметь импортировать и
> где взять их актуальный список». Это исследовательский артефакт вехи 5.5 (роадмап §5.5): фиксирует
> масштаб задачи, авторитетные источники и методологию поддержания списка. Не путать с:
> - как скачать выписку конкретному пользователю → `knowledge/product/bank_statements_user_guide.md`;
> - какие форматы отдаёт каждый банк (матрица + дрейф) → `knowledge/product/banks_statements_reference.md`;
> - как покрыть все банки без парсера под каждый → `docs/universal_statement_parser_strategy.md`.

---

## 1. Сколько банков реально

Рабочая цифра — **около 200 розничных банков** (то, что видел Василий по эквайринговому списку СБП,
подтверждается официальной статистикой).

Опорные факты (все сверяемы по источникам ниже):

- **СБП — 226 банков-участников** (данные ЦБ на 1 марта 2026 и Википедии на 1 апреля 2026). Это
  практически все финучреждения, обслуживающие физлиц: перечень участников СБП — самый близкий к
  «списку банков, с которыми реально сталкивается пользователь». Остальные кредитные организации либо
  узкопрофильные (операторы трансграничных переводов, банки для концентрации активов), либо не дают
  дистанционных сервисов физлицам — им СБП не нужна, и в жизни пользователь их выписку не принесёт.
- **Национальная платёжная система (ЦБ, на 1 января 2026): 31 платёжная система, 353 оператора по
  переводу денежных средств.** Это более широкая юридическая вселенная (включает НКО, платёжные
  сервисы), из которой розничные банки — подмножество.
- **Концентрация.** Системно значимые банки (перечень ведёт ЦБ) держат бо́льшую часть активов рынка;
  порядка 30 участников СБП входят в топ-100 по активам. Практический вывод для покрытия: **20–30
  банков закрывают подавляющее большинство пользователей**, длинный хвост из ~170 — редкие кейсы.

> **Почему цифра «плавает».** Число участников СБП меняется по мере подключения новых банков (225 в
> мае 2025 → 226 в марте–апреле 2026). Поэтому в коде список банков **не хардкодится** — источником
> истины остаётся живой реестр (см. §2). В продукте фиксируем не список имён, а способ его получить.

## 2. Источники истины (живые, авторитетные)

Приоритет — официальные и первичные источники, а не агрегаторы.

1. **Реестр участников СБП — НСПК.** `https://sbp.nspk.ru/participants` (и `https://sbp.nspk.ru/banks`).
   Главный практический список: банки, завершившие сертификационное тестирование и предоставляющие
   сервис клиентам. Ведётся оператором СБП (НСПК). Используем как канонический перечень розничных
   банков.
2. **Банк России — справочник кредитных организаций и реестр лицензий.** `https://cbr.ru` (раздел
   национальной платёжной системы и справочник по кредитным организациям). Полная юридическая
   вселенная: кто вообще имеет банковскую лицензию. Нужен, когда важен юридический статус, а не только
   участие в СБП.
3. **Программный список СБП — API Т-Банка.** `https://securepay.tinkoff.ru/v2/GetQrBankList`
   (`developer.tbank.ru/eacq/api/get-qr-bank-list`). Метод возвращает список банков-участников СБП —
   удобно, если понадобится обновлять справочник автоматически, а не парсить HTML НСПК.

> **Сетевая грабля (см. `docs/infrastructure.md`, `docs/pitfalls.md`).** `sbp.nspk.ru` и `cbr.ru`
> отдают `403` датацентровым IP (DDoS-защита, не баг). Из песочницы/сервера прямой скрейпинг может не
> пройти — на проде тянуть либо через официальный API (п. 3), либо через зеркало/прокси, как это уже
> решено для курса ЦБ (`cbr-xml-daily.ru`). Для разовой выверки списка достаточно ручного экспорта.

## 3. Классификация по приоритету покрытия

Покрываем не «по алфавиту», а по охвату пользователей. Три эшелона:

**Эшелон 1 — розничные гиганты (обязательное покрытие).** Банки, которые есть у большинства
пользователей FINPILOT. По ним нужен надёжный импорт и выверенные инструкции. Ориентировочный список
(уточняется по доле рынка): Сбербанк, Т-Банк (Тинькофф), ВТБ, Альфа-Банк, Газпромбанк, Совкомбанк,
Райффайзенбанк, ПСБ, Росбанк (влит в Т-Банк, операции до 01.01.2025 — через поддержку Т-Бизнеса),
Озон Банк, Яндекс Банк, МТС Банк, ОТП Банк, Почта Банк, Хоум Банк, УБРиР, Ренессанс Банк.
Форматы и статус парсера по каждому — в `banks_statements_reference.md`.

**Эшелон 2 — средние банки из топ-100.** Реже, но встречаются. Покрываются **не отдельным парсером**,
а универсальным движком (CSV/XLSX + 1C), который уже работает с любым банком, где в выписке есть дата
и сумма. Отдельный код — только если банк даёт исключительно нестандартный PDF.

**Эшелон 3 — длинный хвост (~170 банков).** Мелкие/региональные. Явное правило: **не пишем парсер под
каждый**. Опора — universal-парсер + формат 1C (его отдают почти все для юрлиц). Если банк выпал —
пользователю показываем понятное сообщение и предлагаем формат, который движок берёт (CSV/XLSX/1C).

> Обоснование эшелонов и алгоритм «как покрыть 190 без 190 парсеров» — в
> `docs/universal_statement_parser_strategy.md`. Здесь — только вселенная и приоритет.

## 4. Методология поддержания списка

1. **Источник — реестр, не память.** Список банков берётся из НСПК/API (см. §2), а не переписывается
   руками в код. При релизе достаточно свежего экспорта; полный перечень имён в репозитории не
   дублируем (быстро устаревает).
2. **В коде — способность, а не перечень.** `app/api/routes_banks.py::list_banks` отдаёт банки, под
   которые есть выделенный парсер (сейчас 5 + universal + 1C). Универсальный путь принимает файл от
   любого банка — поэтому «неизвестный банк» ≠ «не поддерживается».
3. **Приоритет обновления — по охвату.** Сначала эшелон 1 (доля рынка), потом вширь. Новый банк из
   эшелона 1 → проверить, берёт ли его universal-движок; выделенный парсер — только при
   необходимости.
4. **Дата и источник обязательны.** В `banks_statements_reference.md` каждая строка форматов помечена
   источником и датой — форматы банки меняют, факты надо переваривать заново перед реализацией
   (веха 7).

## 5. Связь с продуктом

- **Роадмап.** §5.5 (этот исследовательский блок) → §7 (реализация покрытия по стратегии из
  `docs/universal_statement_parser_strategy.md`). Пункт роадмапа «Покрытие ~200 банков» — это
  применение стратегии, а не 200 отдельных задач.
- **Ценности** (`knowledge/business/product_philosophy.md`). Широкое покрытие банков = финансовая
  доступность: пользователь любого банка должен суметь занести операции, не вбивая вручную. Это прямо
  влияет на удержание (см. отзывы в `survey_auditory/usability/` — ручной ввод убивает желание вести
  бюджет).
- **Что у нас уже есть.** Мультибанк CSV/XLSX + PDF под 5 банков (Тинькофф, Сбер, ВТБ, Райффайзен;
  Альфа — через universal) + формат 1C. Это фундамент, на котором строится покрытие хвоста.
