# FINPILOT — Пакет документов (болванки-ориентиры)

> ⚠️ **ВАЖНО.** Это **рыбы-ориентиры**, а не юридически выверенные документы, готовые к публикации. Я не юрист. Перед использованием их обязан проверить и доработать профильный юрист по IT/персональным данным. Плейсхолдеры в формате `[...]` нужно заменить. Часть документов (договор с банком) ты не пишешь — их предоставляет банк.

---

## Какие документы нужны и кто их источник

| Документ | Кто составляет | Когда нужен |
|---|---|---|
| Политика обработки ПДн | Ты | Как только собираешь любые данные пользователей |
| Согласие на обработку ПДн | Ты | При регистрации пользователя |
| Согласие на доступ к финансовым данным | Ты | Если подтягиваешь банковские данные |
| Пользовательское соглашение | Ты | При запуске сервиса |
| Уведомление в Роскомнадзор | Ты (подаёшь в РКН) | До начала обработки ПДн |
| Договор/оферта на API | **Банк** | При интеграции с банком |

Ниже — болванки тех четырёх, что пишешь ты.

## 1. Политика обработки персональных данных

> Основание: ст. 18.1 Федерального закона № 152-ФЗ «О персональных данных». Публикуется в открытом доступе (обычно отдельной страницей в приложении/на сайте).

```
ПОЛИТИКА ОБРАБОТКИ ПЕРСОНАЛЬНЫХ ДАННЫХ
в сервисе FINPILOT

1. ОБЩИЕ ПОЛОЖЕНИЯ
1.1. Настоящая Политика определяет порядок обработки персональных данных
     пользователей сервиса FINPILOT (далее — Сервис) и меры по их защите.
1.2. Оператор персональных данных: [ФИО ИП / наименование ООО],
     ИНН [...], адрес [...], e-mail [...].
1.3. Политика разработана в соответствии с Федеральным законом
     от 27.07.2006 № 152-ФЗ «О персональных данных».

2. ОБРАБАТЫВАЕМЫЕ ДАННЫЕ
2.1. Идентификационные данные: имя, адрес электронной почты.
2.2. Финансовые данные, вводимые пользователем или импортируемые
     по его поручению: суммы и категории операций, сведения о доходах,
     расходах, обязательствах, целях и накоплениях.
2.3. Технические данные: данные сессии, журналы доступа.
2.4. Сервис НЕ собирает и НЕ хранит логины и пароли пользователя
     от систем дистанционного банковского обслуживания.

3. ЦЕЛИ ОБРАБОТКИ
3.1. Предоставление функциональности Сервиса: учёт операций,
     расчёт финансовых показателей, формирование рекомендаций.
3.2. Идентификация и аутентификация пользователя.
3.3. Связь с пользователем по вопросам работы Сервиса.

4. ПРАВОВЫЕ ОСНОВАНИЯ
4.1. Согласие пользователя на обработку персональных данных.
4.2. Исполнение соглашения с пользователем (Пользовательское соглашение).

5. ПОРЯДОК И СРОКИ ОБРАБОТКИ
5.1. Обработка осуществляется с использованием средств автоматизации.
5.2. Данные хранятся в течение срока использования Сервиса и [N] месяцев
     после удаления учётной записи, если иное не предусмотрено законом.
5.3. Трансграничная передача данных не осуществляется.
     Данные хранятся на серверах, расположенных на территории РФ.

6. ПРАВА СУБЪЕКТА ПЕРСОНАЛЬНЫХ ДАННЫХ
6.1. Получение информации об обработке своих данных.
6.2. Уточнение, блокирование или удаление данных.
6.3. Отзыв согласия на обработку.
6.4. Обращение в Роскомнадзор за защитой своих прав.

7. МЕРЫ ЗАЩИТЫ
7.1. Шифрование данных при передаче и хранении.
7.2. Разграничение прав доступа, журналирование операций.
7.3. Иные меры в соответствии с разделом «Кибербезопасность».

8. ИЗМЕНЕНИЕ ПОЛИТИКИ
8.1. Актуальная редакция размещается по адресу [...].
8.2. Дата вступления в силу: [дата].
```

## 2. Согласие на обработку персональных данных

> Даётся пользователем активным действием (галочка-чекбокс при регистрации, не предустановленная). Требование 152-ФЗ — согласие должно быть конкретным, информированным, сознательным.

```
СОГЛАСИЕ НА ОБРАБОТКУ ПЕРСОНАЛЬНЫХ ДАННЫХ

Я, пользователь сервиса FINPILOT, свободно, своей волей и в своём
интересе даю согласие [ФИО ИП / наименование ООО], ИНН [...]
(далее — Оператор), на обработку моих персональных данных.

1. Перечень данных: имя, адрес электронной почты, вводимые мной
   финансовые сведения (доходы, расходы, обязательства, цели, операции).

2. Перечень действий: сбор, запись, систематизация, накопление,
   хранение, уточнение, использование, удаление.

3. Цели обработки: предоставление функциональности Сервиса, включая
   учёт операций, расчёт показателей и формирование рекомендаций.

4. Способ обработки: автоматизированная обработка.

5. Срок действия согласия: до достижения целей обработки либо до
   отзыва согласия.

6. Порядок отзыва: путём направления заявления на e-mail [...]
   или через интерфейс Сервиса (удаление учётной записи).

Дата: [автоматически]   Подтверждение: [чекбокс / факт регистрации]
```

## 3. Согласие на доступ к финансовым данным

> Нужно, если Сервис импортирует банковские данные (даже по инициативе пользователя). Это отдельное, явное согласие — данные о финансах чувствительны.

```
СОГЛАСИЕ НА ОБРАБОТКУ ФИНАНСОВЫХ ДАННЫХ

Подтверждая настоящее согласие, я поручаю сервису FINPILOT обработку
финансовых данных, предоставленных мной одним из следующих способов:
   — путём ручного ввода;
   — путём загрузки файла банковской выписки;
   — путём передачи мной токена доступа к API моего счёта
     (для счетов, по которым я являюсь правомочным владельцем).

1. Состав данных: история операций, суммы, даты, назначения и категории
   платежей, остатки по счетам.

2. Цель: исключительно формирование персональных финансовых
   показателей и рекомендаций внутри Сервиса.

3. Я подтверждаю, что:
   — предоставляемые данные относятся к моим собственным счетам;
   — я вправе передавать эти данные Сервису;
   — Сервис не инициирует операций по списанию средств и не имеет
     к ним доступа.

4. Передача третьим лицам: данные не передаются третьим лицам,
   за исключением случаев, предусмотренных законом.

5. Отзыв: я могу в любой момент отозвать согласие и удалить
   импортированные данные через интерфейс Сервиса.

Дата: [автоматически]   Подтверждение: [чекбокс]
```

```

1. ТЕРМИНЫ
1.1. Сервис — программный продукт FINPILOT.
1.2. Оператор — [ФИО ИП / наименование ООО], ИНН [...].
1.3. Пользователь — физическое лицо, использующее Сервис.

2. ПРЕДМЕТ
2.1. Оператор предоставляет Пользователю доступ к функциональности
     Сервиса: учёт личных финансов, расчёт показателей, формирование
     рекомендаций по распределению свободных средств.

3. ХАРАКТЕР РЕКОМЕНДАЦИЙ  ← ключевой раздел для финтеха
3.1. Рекомендации Сервиса носят информационно-справочный характер.
3.2. Рекомендации НЕ являются индивидуальной инвестиционной
     рекомендацией в значении Федерального закона № 39-ФЗ
     «О рынке ценных бумаг», финансовой, налоговой или юридической
     консультацией.
3.3. Решения, принимаемые на основе рекомендаций, Пользователь
     принимает самостоятельно и на свой риск.

4. ПРАВА И ОБЯЗАННОСТИ
4.1. Пользователь обязуется предоставлять достоверные данные.
4.2. Пользователь самостоятельно отвечает за сохранность доступа

5. ОГРАНИЧЕНИЕ ОТВЕТСТВЕННОСТИ
5.1. Оператор не несёт ответственности за решения Пользователя,
     принятые на основе предоставленной Сервисом информации.

6. ПЕРСОНАЛЬНЫЕ ДАННЫЕ
6.1. Обработка данных регулируется Политикой обработки персональных
     данных, размещённой по адресу [...].

7. ЗАКЛЮЧИТЕЛЬНЫЕ ПОЛОЖЕНИЯ
7.1. Соглашение вступает в силу с момента начала использования Сервиса.
```

## Порядок действий (если идёшь в коммерциализацию)

1. Регистрируешь юрлицо/ИП → получаешь реквизиты для подстановки в документы.
2. Юрист дорабатывает все четыре документа под твою конкретную модель.
3. Подаёшь уведомление в Роскомнадзор об обработке ПДн (до начала обработки).
4. Публикуешь Политику и Пользовательское соглашение в приложении.
5. Внедряешь согласия как явные чекбоксы при регистрации/импорте.
