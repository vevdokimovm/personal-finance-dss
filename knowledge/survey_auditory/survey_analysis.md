# FINPILOT Survey Analysis

Пайплайн анализа пользовательского исследования FINPILOT: от сырой выгрузки
Google Forms до HTML-отчёта с графиками и машиночитаемых результатов.

Анализирует 385 анкет (322 валидных после контроля внимательности) по методологии
количественного и качественного анализа: описательная статистика, проверка гипотез
с размерами эффекта и поправкой на множественность, поведенческая валидация
(stated vs revealed), факторная диагностика критериев, продуктово-бизнесовый слой
(WTP, opportunity scoring, RICE, доверие к ИИ) и кодирование открытых ответов.

## Структура

```
finpilot_survey/
├── config.py              # маппинг вопросов, шкалы, варианты мультиселектов
├── data.py                # загрузка, фильтр выборки, кодирование, мультиселекты
├── stats_utils.py         # размеры эффекта, тесты, поправка Холма, ДИ
├── analyzers_quant.py     # описательная, портрет ЦА, сегменты, гипотезы,
│                          #   критерии, ранжирования
├── analyzers_product.py   # поведенческий, бизнес и качественный анализ
├── visualization.py       # генерация графиков (matplotlib → PNG)
├── report.py              # сборка HTML-отчёта
└── pipeline.py            # оркестратор
run_analysis.py            # точка входа
```

## Запуск

```bash
python -m pip install -r requirements.txt
python run_analysis.py --data path/to/survey.xlsx --out output
```

По умолчанию данные берутся из пути в `config.py`, результаты пишутся в `output/`.

## Что на выходе

```
output/
├── finpilot_report.html   # отчёт (открыть в браузере)
├── charts/                # 14 графиков PNG
└── results.json           # все результаты в машиночитаемом виде
```

## Методологические решения

- **Валидная выборка (322).** Из 385 удалены 63 анкеты, не прошедшие
  attention-check («выберите 7 звёзд»). Прошедшие проверку и отвечавшие до её
  добавления — оставлены.
- **Переменный N.** Анкета дополнялась в ходе сбора, закрытые вопросы были
  обязательными — поэтому у каждого показателя свой N (это не «отвал»).
  Репрезентативность поздней когорты проверяется в `AudienceProfiler`.
- **Размер эффекта обязателен.** Ни один p-value не репортится без размера
  эффекта и поправки Холма на множественные сравнения.
- **Малый N помечается.** Блоки на ≈19 ответах (критерии, кейс, доверие к ИИ)
  везде маркируются как предварительные.
- **Чего нет — не выдумывается.** Retention, LTV/CAC, когортный анализ требуют
  работающего продукта с логами и на данных опроса не считаются.
