# Эталонные профили (ресинхронизация через движок v4.16.9)

Шесть эталонных профилей, **пересчитанных реальным движком FINPILOT v4.16.9** (66 альтернатив,
шаг 10%, stock-based ликвидность). Значения получены вызовом `/api/planning/calculate` на
демо-данных (профиль риска по умолчанию — сбалансированный, `r_bench = 0.14`). Это канонический
эталон: ожидаемые `a*` по определению совпадают с выходом движка и используются для валидации
вместо устаревших расчётов.

> **Обновлено под модель v3.0.0.** Прежняя редакция этого файла приводила значения движка v2.1.0
> (flow-based `Lt = Rt/(E+ΣP)`, шаг 20% → 21 альтернатива). С переходом на v3.0.0 ликвидность стала
> **stock-based** (`Lt = B_liq/E`, месяцы автономии), дискретизация — шаг 10% (66 альтернатив).
> Поэтому `Lt` теперь в месяцах и равен 0 у кейсов без ликвидной позиции, а оптимум сместился в
> сторону пополнения резерва (ликвидность стала независимым от ресурса критерием). Это ожидаемо.

> Демо-кейсы загружаются в приложении (боковая панель) или через `POST /api/demo/load?case=<имя>`.

## Пояснения к ключевым случаям

- **mikhail — отказ (fail-loud).** При Rt = −3 000 ₽ свободных денег нет: расходы и платежи
  превышают доход. Движок **не выдаёт рекомендацию** (0 допустимых альтернатив), а сигнализирует о
  структурной проблеме. Это поведение по принципу fail-loud, а не сбой.
- **olga — досрочка.** Единственный профиль, где оптимум — направить всё на досрочное погашение:
  ставка кредита выше порога `r_bench = 0.14`, и Avalanche делает досрочку экономически выгоднее
  накоплений. U = 0.50.
- **anna / dmitriy / igor / viktor — резерв.** При сбалансированном профиле (вес ликвидности 0.30 —
  наибольший) оптимум — пополнить резерв: это поднимает stock-based `Lt` (месяцы автономии) и **не
  снижает** `Rt` (деньги в резерве остаются доступными). Раньше, при flow-based `Lt`, эти кейсы шли в
  цели — но тогда ликвидность была линейна ресурсу и не работала как отдельный критерий. Это и есть
  видимый эффект перехода на stock-based ликвидность в v3.0.0.
- **igor / viktor — 11 альтернатив.** Без кредитов переменная досрочки вырождается, остаётся
  распределение между резервом и целями → 11 комбинаций при шаге 10% (вместо 66).

## Как воспроизвести

```bash
# на каждом кейсе (гостевой режим)
curl -X POST "http://localhost:8000/api/demo/load?case=anna"
curl -X POST "http://localhost:8000/api/planning/calculate" -H "Content-Type: application/json" -d '{}'
```

Поле `best` ответа — рекомендованное `a*` (`x_obligations` / `x_reserve` / `x_goals`, `utility`);
`indicators` — базовые показатели (`Rt`, `Lt`, `Dt`, `BLR`, …); `admissible_count` — число прошедших
фильтр альтернатив.
