# Реестр инцидентов FINPILOT

Краткие сводки по каждому **инциденту** — был дефект в продуктовом коде и/или влияние на
пользователя/систему. Расследования, где дефекта в коде НЕ оказалось (внешняя причина, артефакт
теста, недопонимание), — в отдельном парном реестре `docs/investigations_summary.md`.

Полные разборы (post-mortem) живут ОТДЕЛЬНЫМИ файлами вне репозитория (правило §12). Формат разбора —
`docs/incident_postmortem_guide.md`; шаблон карточки дефекта — `docs/bug_report_template.md`. В каждой
записи — не только корень, но и **решение / шаги** (а если не закрыто — что опробовано и что дальше).

---

### INC-CRUD-DELETE — удаление/восстановление obligations/goals/transactions

- **Суть:** удаление obligations было сломано несколько версий при зелёных юнит-тестах; restore (undo) добавлен позже; E2E на базовые CRUD не было.
- **Корень:** юнит проверяет функцию в изоляции, но не разрыв между UI и API — кнопка/`data-*`-атрибут/эндпоинт могли разойтись незаметно.
- **Решение / шаги:** добавлены 6 CRUD-E2E через браузер (create/delete/restore для obligations, goals, transactions, v4.16.28), зелёные на chromium; введено правило §13 (базовые CRUD ОБЯЗАНЫ иметь E2E). Бэк restore (симметричный soft-delete) закрыт ранее в v4.16.18 (миграция 0018).
- **Статус:** закрыт. Полный разбор — у владельца (правило §12).

### BUG-024 / BUG-025 — дефекты, разобранные в E2E-постмортеме

- **Суть:** дефекты, найденные при сквозном E2E-разборе (CSRF/CORS + флаги браузера); фикс в несколько строк искался долго (симптом `-1` приняли за «смерть песочницы»).
- **Корень:** см. полный разбор (CSRF/CORS-конфигурация + флаги запуска браузера).
- **Решение / шаги:** дефекты исправлены; урок вынесен в антипаттерны `incident_postmortem_guide.md` — не списывать на «невозможно», не перебрав каналы.
- **Статус:** закрыт. Полный разбор — у владельца (правило §12).

> Как добавлять: при закрытии инцидента с дефектом кода — блок сюда (суть + корень + **решение/шаги**;
> если не закрыто — что опробовано / почему / что дальше) + полный разбор отдельным файлом владельцу
> (правило §12). Расследования без дефекта продуктового кода — в `docs/investigations_summary.md`.
