# Реестр инцидентов FINPILOT

Краткие сводки по каждому **инциденту** — был дефект в продуктовом коде и/или влияние на
пользователя/систему. Расследования, где дефекта в коде НЕ оказалось (внешняя причина, артефакт
теста, недопонимание), — в отдельном парном реестре `docs/investigations_summary.md`.

Полные разборы (post-mortem) живут ОТДЕЛЬНЫМИ файлами вне репозитория (правило §12). Формат разбора —
`docs/incident_postmortem.md`; шаблон карточки дефекта — `docs/bug_report_template.md`. В каждой
записи — не только корень, но и **решение / шаги** (а если не закрыто — что опробовано и что дальше).

---

### INC-CRUD-DELETE — удаление/восстановление obligations/goals/transactions

- **Суть:** удаление obligations было сломано несколько версий при зелёных юнит-тестах; restore (undo) добавлен позже; E2E на базовые CRUD не было.
- **Корень:** юнит проверяет функцию в изоляции, но не разрыв между UI и API — кнопка/`data-*`-атрибут/эндпоинт могли разойтись незаметно.
- **Решение / шаги:** добавлены 6 CRUD-E2E через браузер (create/delete/restore для obligations, goals, transactions, v4.16.28), зелёные на chromium; введено правило §13 (базовые CRUD ОБЯЗАНЫ иметь E2E). Бэк restore (симметричный soft-delete) закрыт ранее в v4.16.18 (миграция 0018).
- **Статус:** закрыт. Полный разбор — `FINPILOT_crud_delete_incident.md` (вне репо).

### BUG-024 / BUG-025 — дефекты, разобранные в E2E-постмортеме

- **Суть:** дефекты, найденные при сквозном E2E-разборе (CSRF/CORS + флаги браузера); фикс в несколько строк искался долго (симптом `-1` приняли за «смерть песочницы»).
- **Корень:** см. полный разбор (CSRF/CORS-конфигурация + флаги запуска браузера).
- **Решение / шаги:** дефекты исправлены; урок вынесен в антипаттерны `incident_postmortem.md` — не списывать на «невозможно», не перебрав каналы.
- **Статус:** закрыт. Полный разбор — `FINPILOT_E2E_incident_postmortem.md` (вне репо).

### INC-A11Y-DEADTEST — live-a11y тест молча скипался (мёртвый axe из-за CSP)

- **Суть:** предрелизный live-a11y тест (`tests/full/test_a11y_axe.py`, axe-core) скипался во ВСЕХ прогонах (и в CI), потому что axe грузился с внешнего CDN, а CSP приложения (`script-src 'self' 'unsafe-inline'`) внешние скрипты режет. Прогон был «зелёным» за счёт `skip`, а не `pass` — тест фактически не проверял доступность ни на одной странице. Near miss (приложение не в проде).
- **Корень:** тест зависел от внешнего ресурса (CDN), недоступного из-за СОБСТВЕННОЙ же security-политики приложения, И деградация маскировалась молчаливым скипом вместо падения. 5 Whys — в полном разборе.
- **Решение / шаги:** axe-core 4.12.1 вендорен (`tests/full/vendor/axe.min.js`) + inline-инъекция (`add_script_tag(content=...)`, проходит по `'unsafe-inline'`, не зависит от сети) — v4.16.32. Тест расширен до 14 страниц × 2 темы, critical=0 жёстко + serious-ratchet. Починенный тест сразу вскрыл реальный дефект `nested-interactive` (см. INV-A11Y-NESTED / батч доступности).
- **Статус:** тест-инфраструктура закрыта (v4.16.32). Вскрытые дефекты доступности — в батче доступности. Полный разбор — `FINPILOT_a11y_dead_test_incident.md` (вне репо). Action item на будущее: критичный тест не должен молча скипаться при недоступности обязательного ресурса (вендорить или фейлить) + сделать число скипов видимым сигналом.
