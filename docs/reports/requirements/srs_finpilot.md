# SRS — FINPILOT (Software Requirements Specification)

Заполненная спецификация требований по шаблону `knowledge/guides/templates/srs_guide.md`.
Каноничная математическая модель — **v3.0.0** (`docs/math_model_v3_0_0.md`); v2.x — legacy, не использовать.

## 1. Назначение
FINPILOT — алгоритмическая СППР (система поддержки принятия решений) для персональных финансов:
по финансовому снимку пользователя рекомендует, как распределить свободные средства между
погашением долга, резервом, целями и тратами, с прозрачным объяснением.

## 2. Функциональные требования
- **FR-1. Импорт выписок.** Приём CSV / XLSX / PDF / 1C (`1CClientBankExchange`); авто-распознавание
  доходов и расходов (`app/ingestion/`). Пользовательский гайд — `knowledge/product/bank_statements_user_guide.md`.
- **FR-2. Расчёт снимка.** Ликвидность `Lt = B_liq / Σe` (stock-based, месяцы автономии); ресурс
  `Rt = CF − ΣP` (поток, баланс не входит).
- **FR-3. Многокритериальная оптимизация.** SAW по критериям (доходность, ликвидность, снижение
  долга, безопасность) на `|A| = 66` альтернативах (шаг дискретизации 10%).
- **FR-4. Профили риска.** 5 профилей; веса целей неубывающие (плато 0.20 между профилями 2–3).
- **FR-5. Фильтр долга.** Avalanche — отсев «дешёвого» долга.
- **FR-6. Прогноз.** SES + Монте-Карло; доверительный интервал 80% (p10–p90).
- **FR-7. Бенчмарк доходности.** `r_bench = ключевая ставка ЦБ × (1 − НДФЛ)`; фолбэк `0.14` при
  недоступности источника.
- **FR-8. Рекомендация с объяснением.** Конкретное распределение + цифры/логика.

## 3. Бизнес-правила (инварианты)
- **Жёсткие:** `Rt ≥ 0`; `ПДН ≤ 0.40`.
- **Мягкие:** `L_min = 0` (по умолчанию выключен).

## 4. Нефункциональные требования
- **Безопасность / 152-ФЗ:** аутентификация JWT + revocation, шифрование в покое (MultiFernet),
  household-модель для удаления ПДн. Срез — `docs/reports/security/finpilot_security_audit.md`.
- **Данные:** PostgreSQL (прод), SQLite (тесты) — ADR-004. Схемные изменения проверяются на PostgreSQL.
- **Платформа:** FastAPI · SQLAlchemy 2.0 · Alembic · Pydantic v2 · Jinja2 + vanilla JS · Python 3.12.
- **Качество:** покрытие ≥ 90%; property-based тесты на модель (Hypothesis). Стратегия —
  `docs/reports/testing/test_strategy_report.md`.

## 5. Ограничения РФ
- `cbr.ru` отдаёт 403 датацентровым IP → прод-зеркало `cbr-xml-daily.ru`.
- Ключевая ставка ЦБ кэшируется (`cbr_key_rate_cache`), SOAP-метод `KeyRate` регистрозависим.

## 6. Вне скоупа (текущей вехи)
Open Banking RU (2026+, регуляторика), Telegram-бот, семейные бюджеты (P3.7, архитектура решена).
