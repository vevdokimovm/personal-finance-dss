# docs/reports/ — отчёты проекта

Единый дом для всех типов отчётов. Каждый вид — в своей папке.

## Структура
- **adr/** — Architecture Decision Records (принятые архитектурные решения).
- **incidents/** — инциденты: дефекты продукта с влиянием на пользователя, post-mortem'ы.
- **investigations/** — расследования: неочевидная причина без дефекта в коде.
- **merges/** — merge-манифесты и отчёты о слияниях параллельных веток + гайд по форку/мёржу.

## Реестры и методология (корень reports/)
- `incidents_summary.md`, `investigations_summary.md` — сводные реестры.
- `incident_postmortem_guide.md`, `investigation_report_guide.md` — как писать отчёты.
- `bug_report_template.md`, `report_types.md` — шаблон баг-репорта и таксономия отчётов.

## Таксономия (кратко)
Инцидент = дефект с impact на юзера. Расследование = разобрались в неочевидной причине,
дефекта в коде нет. Bug card = найден новый дефект в коде. ADR = зафиксированное
архитектурное решение.
