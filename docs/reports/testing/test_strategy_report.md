# Тест-отчёт: стратегия тестирования FINPILOT

- **Версия:** срез на `v5.3.0`.
- **Тип:** обзор тестовой стратегии и инфраструктуры (не разовый прогон).

## 1. Гейты качества (блокирующие)
- **mypy** — ноль ошибок типов; **flake8** — ноль нарушений.
- **Покрытие ≥ 90%** — жёсткий гейт.
- **bandit**, **pip-audit** — безопасность кода и зависимостей.

## 2. Уровни тестов
- **Unit** — быстрый цикл (SQLite).
- **Матрица SQLite + PostgreSQL** — обязательна для схемных изменений (SQLite молча игнорирует FK).
- **Property-based (Hypothesis)** — 26 тестов на математическую модель (инварианты `Rt≥0`, `ПДН≤0.40`,
  монотонность весов целей и пр.).
- **E2E (Playwright, Chromium)** — CRUD-флоу обязаны иметь E2E; добавляются вместе с unit при
  изменениях фронта/API. Firefox/WebKit в песочнице недоступны.

## 3. Нагрузочное тестирование
- **Locust** (`loadtest/locustfile.py`) — сценарии нагрузки на API. Запуск и метрики — `loadtest/README.md`.

## 4. CI-тиры
- **Fast** — каждый push/PR: SQLite + PostgreSQL, гейт покрытия 90%.
- **Full** — теги/ручной: мультибраузерный E2E + визуальная регрессия + live-a11y + security.
- **Deep** — cron/ручной: stress-property + мутационное тестирование.

## 5. Практики и грабли (из истории)
- Вывод тестов — через `--junitxml` (парсить XML + exit code), не `tail` по логу (итоговая строка не
  флашится надёжно) — `docs/pitfalls.md`.
- Механика полного fast-прогона — `docs/sandbox_runbook.md`; что гонять и когда —
  `docs/test_run_optimization.md`.
- Ложная тревога «баг мат-модели» оказалась E2E race condition — `docs/reports/incidents/mathmodel_falsealarm_incident.md`.
