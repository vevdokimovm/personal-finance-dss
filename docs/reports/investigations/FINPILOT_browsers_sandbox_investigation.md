## 1. Вопрос / подозрение

При попытке расширить E2E с одного движка (Chromium) на все три (Chromium / Firefox / WebKit)
тесты на firefox и webkit не запускались. **Страшная гипотеза:** сломана наша E2E-инфраструктура —
неправильный `tests/e2e/conftest.py`, неверные флаги Playwright или мультибраузерный прогон в
принципе настроен криво. Если так — все мультибраузерные тесты пришлось бы переписывать, а доверие
к E2E-слою падало бы.

## 2. Краткий вывод (TL;DR)

**Дефекта в нашем коде/конфиге нет.** Браузерные бинари firefox/webkit физически отсутствуют,
потому что Playwright качает их со своего Azure-CDN (`cdn.playwright.dev` /
`playwright.download.prss.microsoft.com`) — этот CDN **вне сетевого allowlist песочницы** (403; в
allowlist только `github.com`, `pypi.org`, `npmjs.com`, зеркала `ubuntu.com` и т.п.). Chromium
работает, потому что был **предустановлен** в образе, а не скачан. Тесты браузер-агностичны и
корректны; на машине с доступом к CDN (локально / в CI) мультибраузер пройдёт теми же командами
`--browser firefox --browser webkit`. **Все 4 альтернативных канала доставки бинарей перебраны и
закрыты (§7) — обойти в песочнице нечем.**

## 3. Гипотезы

- **A** — неверный конфиг E2E / `conftest.py` / флаги Playwright (наш код).
- **B** — команда установки браузеров не отработала, бинарей нет.
- **C** — не хватает системных библиотек (webkit капризен к либам).
- **D** — сеть/CDN заблокированы, браузеры не скачиваются (внешнее).

## 4. Ход расследования (первичный, аккаунт S)

1. **`playwright install firefox webkit`** → команда завершилась без явной ошибки (на этапе
   install), что поначалу указывало мимо D.
2. **Прогон одного теста `--browser firefox`** → ошибка:
   `BrowserType.launch: Executable doesn't exist at /opt/pw-browsers/firefox-1495/firefox/firefox`.
   Бинаря НЕТ, хотя install «прошёл». Это **отбрасывает A**: конфиг ни при чём — браузер физически
   отсутствует, до запуска дело не доходит.
3. **Осмотр `/opt/pw-browsers/`** → присутствует только `chromium-1194` (+ headless shell, ffmpeg);
   firefox/webkit отсутствуют. Подтверждает B (бинарей нет) — но нужно понять *почему*.
4. **Прогон `--browser webkit`** → `EXIT 127` (исполняемого нет). Та же картина, что у firefox.
5. **Трассировка источника (решающий шаг).** Playwright тянет browser builds со своего Azure-CDN.
   Сверка с сетевым allowlist песочницы: CDN туда **не входит**. → подтверждает **D**: скачивание
   блокируется на уровне сети. Chromium присутствует не потому, что скачался, а потому что
   предустановлен в образе песочницы.
6. **`playwright install-deps firefox webkit`** → провал на `apt` (сломанный `nodesource`-репозиторий).
   Это отдельная проблема (системные либы для webkit), но вторичная: без бинаря (шаг 5) либы всё
   равно не помогут.
7. **Контрольная проверка.** Прогон на chromium → **20 E2E зелёные** (5 dashboard + 5 planning +
   4 auth + 6 CRUD). Значит сама E2E-инфраструктура исправна — проблема не в ней.

## 5. Корень / вывод

**Внешнее ограничение окружения:** CDN Playwright (`cdn.playwright.dev` /
`playwright.download.prss.microsoft.com`) вне сетевого allowlist песочницы, поэтому бинари
firefox/webkit нельзя скачать. Это **не дефект нашего кода** и **не ошибка конфигурации тестов**.
Граница «от нас не зависит»: сетевая политика песочницы — как и блок `cbr.ru` для датацентровых IP.
Chromium доступен только потому, что предустановлен в образе.

## 6. Что это НЕ (снятые подозрения)

- **НЕ** ошибка в `tests/e2e/conftest.py` или флагах Playwright — конфиг корректен, на chromium всё
  работает.
- **НЕ** неверные селекторы или тесты — те же тесты зелёные на chromium.
- **НЕ** требует правки продуктового кода — мультибраузерность включается флагом `--browser`,
  тесты браузер-агностичны.
- **НЕ** «E2E-слой ненадёжен» — 20 тестов зелёные на доступном движке.

## 7. Перебор каналов — хвост §11 закрыт (аккаунт V)

Первичный разбор оставил 4 канала «что ещё попробовать, прежде чем объявлять окончательно». Все
перебраны эмпирически — каждый закрыт конкретной командой:

| Канал | Что проверяли | Результат | Доказательство |
|---|---|---|---|
| **(а) GitHub releases** | Раздаёт ли Microsoft сборки браузеров на GitHub | ✗ нет | `PLAYWRIGHT_DOWNLOAD_HOST=https://github.com playwright install firefox` → URL `github.com/builds/firefox/1495/firefox-ubuntu-24.04.zip` → **404 Not Found** (хост доступен, билда по пути нет). GitHub API проверить не дал — rate-limit на общем egress-IP. |
| **(б) npm-пакет** | Содержит ли npm-пакет `playwright` бинари | ✗ нет | Тарбол `playwright@1.56.0` с `registry.npmjs.org` = 866 КБ, 170 файлов, ноль `*-ubuntu-*.zip`. npm `playwright install` тянет браузеры с того же Azure-CDN — обхода нет. |
| **(в) системный firefox** | Подсунуть стоковый firefox в `p.firefox.launch()` | ✗ несовместим | `executable_path` на сторонний бинарь падает мгновенно; `p.firefox.launch()` жёстко привязан к managed-бинарю `/opt/pw-browsers/firefox-1495/...`. Playwright firefox требует juggler-патченный билд; стоковый Mozilla firefox не подходит. Для webkit на Linux системного аналога, работающего с Playwright, нет вовсе. |
| **(г) apt** | Даёт ли apt бинарь браузера | ✗ только либы | `firefox-esr` — `no installation candidate` (без `apt-get update`); сам `apt-get update` падает на стороннем `nodesource`-репо (403), ubuntu-репы (`archive/security.ubuntu.com`) живы. Принципиально: apt поставляет runtime-ЛИБЫ (`playwright install-deps`), а не бинарь браузера. Либы необходимы, но недостаточны. |

**Почему ни один не сработал:** Playwright раздаёт кастомные (juggler/WebKit-патченные) сборки
**исключительно** через свой CDN: на GitHub их нет, в npm-пакете нет, системными браузерами не
заменить. → Это окончательно внешняя граница, а не «пока не разобрались».

**Что остаётся (не обход, а условие):** если allowlist песочницы расширят на CDN Playwright —
`playwright install firefox webkit` поднимет их без единой правки в тестах. До тех пор: chromium в
песочнице; мультибраузер (chromium + firefox + webkit) — в CI (полный тир `full`), где CDN доступен.

## 8. Уроки

- Отсутствующий бинарь браузера **выглядел** как баг тест-инфраструктуры, а оказался сетевой
  политикой окружения. Решающий шаг — проверить **реальный источник скачивания** (URL CDN) и
  сверить с allowlist, а не крутить конфиг тестов.
- Контрольный прогон на заведомо рабочем движке (chromium) сразу отделил «инфраструктура цела» от
  «среда не даёт скачать» — это дёшево и снимает половину гипотез.
- **Правило §11 в действии:** вывод «внешнее ограничение» не финализируется, пока не перебраны все
  каналы обхода. S дошёл до вывода и честно оставил хвост; V перебрал хвост и закрыл его командами.
  До перебора корректная формулировка была «через `playwright install` в этой песочнице не выходит»,
  после — «обойти нечем: бинари существуют только на CDN, который недоступен».
- Признак чистого investigation report: продуктовый код и тест-конфиг **сняты с подозрения**
  доказательно; настоящая причина названа с границей «от нас не зависит».
