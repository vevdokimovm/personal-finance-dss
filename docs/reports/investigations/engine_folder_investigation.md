# Расследование: нужна ли папка `engine/`

- **Дата:** 2026-07-01
- **Тип:** расследование (дефекта в коде нет)
- **Триггер:** вопрос «где-то упоминалась папка Engine — кто, зачем, нужна ли».

## Подозрение
В материалах фигурирует путь `engine/core.py`, будто существует отдельная папка `engine/`.

## Что проверено (эмпирически, по репозиторию)
1. Поиск по коду и докам: `grep -rIn -i engine`.
2. Реальная структура `app/`: папки `engine/` на верхнем уровне **нет**; есть `app/core/`.
3. Тесты `tests/test_international.py` импортируют движок из `app.ingestion`:
   `from app.ingestion.engine import CoreFinanceEngine`,
   `from app.ingestion.contracts import FinanceEngine`.
4. Источник путаницы — `knowledge/business/international_expansion_logic.md:188`:
   комментарий `# engine/core.py (УЖЕ существует — меняется только вход)`.

## Корень
«Engine» реально существует, но **не как папка `engine/`**:
- **`app/ingestion/engine.py`** — класс `CoreFinanceEngine` (расчётный движок: SAW + Avalanche + Монте-Карло);
- **`app/ingestion/contracts.py`** — протокол `FinanceEngine` (источнику-агностичный контракт);
- **`app/core/`** — ядро расчётов.

Запись `engine/core.py` в бизнес-доке — **концептуальный псевдо-путь** (иллюстрация к логике
интернационализации), а не фактическая структура. Отсюда ложное впечатление про отдельную папку.

## Вывод
**Папка `engine/` не нужна** — движок уже реализован в `app/ingestion/` (+ `app/core/`) и покрыт
тестами. Создавать её означало бы дублировать существующее.

## Что дальше (необязательно)
Для устранения путаницы — поправить псевдо-путь в `international_expansion_logic.md:188`
с `engine/core.py` на фактический `app/ingestion/engine.py`. Это правка бизнес-иллюстрации, не кода.
