# Security-аудит FINPILOT — итог вехи 4 (раздел 4.4)

- **Дата / версия:** 2026-07-01, срез на `v4.30.2` (закрытие вехи 4).
- **Скоуп:** бэкенд-безопасность — аутентификация, сессии, шифрование, ПДн; раздел ROADMAP §4.4.
- **Метод:** обзор кода + история релизов (`CHANGELOG.md`, WATCHLOG §3) + `bandit`, `pip-audit`;
  чек-лист — `knowledge/guides/cybersecurity_finpilot_checklist.md`.

## 1. Резюме
Раздел 4.4 (cybersecurity hardening) **закрыт полностью**. Реализованы: инвалидация JWT (точечная и
массовая), гашение сессий при смене/сбросе пароля, ротация ключа шифрования (MultiFernet),
старт-гард ключа, MFA/TOTP, набор security-регрессий. Критичных открытых дыр по итогам вехи нет;
остаточные пункты — инфраструктурного характера (см. §4).

## 2. По доменам
| Домен | Статус | Комментарий |
|---|---|---|
| Аутентификация и сессии | **ok** | JWT + revocation (`revoked_tokens`, `users.tokens_valid_since`); `POST /api/auth/logout` (точечно по `jti`), `/logout-all` (сдвиг cutoff). Смена/сброс пароля вызывает `bump_tokens_valid_since` — старые сессии гаснут. |
| Авторизация / контроль доступа | **ok** | Ресурсы ограничены владельцем; household-модель (`household_id` + `created_by` + `is_private`). |
| Защита данных (шифрование) | **ok** | Шифрование «в покое» через Fernet → **MultiFernet** (ротация ключей, `v4.28.0`); старт-гард наличия ключа (`v4.29.0`). |
| Защита от типовых атак | **частично** | CSP присутствует, но с `unsafe-inline` — план перехода на nonce-политику. |
| Журналирование | ok | Аудит-поля (naive-UTC, ADR-002); события auth логируются. |
| ПДн / 152-ФЗ | **ok (с оговоркой)** | `created_by` обязателен для удаления данных по 152-ФЗ; `is_private` для персональных данных внутри общего household. Открытый вопрос — data-residency при интеграции Claude API (не решён). |
| MFA | **ok** | MFA/TOTP реализован (`v4.30.0`). |

## 3. Находки (по итогам вехи — закрытые)
1. **Старые сессии переживали смену пароля** — закрыто в `v4.27.0` (симметричная инвалидация всех сессий).
2. **Ключ шифрования не ротировался** — закрыто в `v4.28.0` (MultiFernet).
3. **Отсутствие старт-гарда ключа** (риск запуска без ключа) — закрыто в `v4.29.0`.
4. **Same-second edge case JWT** (iat секундной гранулярности) — принят как осознанный industry-tradeoff,
   задокументирован в `app/database/revocation.py` (строгое `<` по `tokens_valid_since`).

## 4. Известные пробелы и план (инфраструктура, вне бэк-кода вехи 4)
- **Rate limiting — in-memory**, уязвим на multi-instance. План: Redis-backed (следующий в очереди §4.4-инфра).
- **CSP `unsafe-inline`** → nonce-based policy.
- Оба пункта — инфраструктурные, к деплою; не блокируют закрытие бэк-части 4.4.

## 5. Инструменты и прогон
- `bandit` — статический анализ Python (в CI security-тира).
- `pip-audit` — уязвимости зависимостей (см. также `docs/reports/incidents/dependencies_cve_postmortem.md`).
- Security-регрессии (`v4.29.0`) — защитная сетка против повторного открытия закрытых дыр.
