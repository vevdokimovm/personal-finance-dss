## Правило версий (важно — читать первым)

**Вехи проекта нумеруются как мажорные версии (v4 → v9), и это СОЗНАТЕЛЬНО НЕ классический
SemVer.** В строгом SemVer `MAJOR` = слом обратной совместимости контракта. Здесь мажор =
**крупный этап проекта** (бэкенд / организация / фронт / тестирование / юр / деплой), даже если
этап контракт не ломает. Так нагляднее для одиночной разработки на 4 аккаунтах. Решение принято
Василием осознанно.

Внутри вехи — обычная механика: фича = MINOR (`v4.19.0`), фикс/мелочь = PATCH (`v4.18.2`).
Каждый батч: bump `APP_VERSION` в `app/config.py` + запись в `CHANGELOG.md`.

| Веха | Версия | Тема | Статус |
|---|---|---|---|
| 4 | `v4.x.x` | **Завершение бэкенда + техдолг** | В работе |
| 5 | `v5.0.0` | Организационное (репозитории, доки, бизнес, исследования, методички, банки) | Запланировано |

| 7 | `v7.0.0` | Тестирование (мат-модель 1000+ кейсов + независимый эксперт + импорт + банки) | Запланировано |

| 9 | `v9.0.0` | Деплой на сервер | Запланировано |

Порядок строгий: бэк → организация → фронт → тестирование → юр → деплой.

## 4.2. Баги бэка (из заметок Василия — давние, закрываем)

- [x] **Ставка ЦБ не подтягивается.** Закрыто в **v4.24.0** (инцидент INC-CBR-KEYRATE-PARAM). Корень
  оказался НЕ внешним 403, а дефектом кода: cbr.ru ASMX чувствителен к регистру — метод `KeyRate`
  ждёт `<ToDate>` (заглавная T), в коде была строчная `<toDate>` → параметр не распознан → ToDate=
  DateTime.MinValue → пустой диапазон → ноль строк KR → вечный fallback при живом cbr.ru. Фикс
  регистра + durable БД-кэш last-known-good (`cbr_key_rate_cache`, миграция 0026, переживает прод-403
  на датацентровом IP) + устойчивый парсер + самодиагностика (сниппет ответа в `detail`) +
  `refresh_key_rate()` для прогрева. Подтверждено первоисточником cbr.ru. Живая верификация на РФ-IP —
  у владельца. Маскировалось историей INV-CBR-403 (graceful fallback скрыл реальный баг).

- [x] **Письма не приходят.** Закрыто в **v4.25.0** — но это **расследование**, не дефект
  (INV-EMAIL-FIRE-AND-FORGET). Код формирования/отправки корректен (транспорт покрыт 9 тестами).
  «Пусто на почте» в dev — намеренный no-op без SMTP-конфига; фронт честно отдаёт ссылку
  подтверждения прямо в тосте, а не имитирует отправку. Реальный пробел: фоновая отправка
  (`BackgroundTasks`) теряла результат — сбой SMTP на проде был бы виден только в текстовом логе.
  Добавлена наблюдаемая обёртка `email_dispatch.py`: каждая отправка пишет событие
  `email_sent`/`email_skipped`/`email_failed` в `events` (домен вместо адреса — PII). Теперь при
  запуске на проде с реальным SMTP статус отправки виден в аналитике. Живая верификация с реальным
  SMTP — у владельца (P0.2, веха деплоя).

## 4.4. Кибербезопасность до прод-уровня

> Состояние: бэкенд-минимум OWASP **закрыт** — bcrypt-хеши (`services/security.py`), Fernet-шифрование
> «в покое» (`EncryptedString`), JWT + account lockout, 4 middleware (CSRF/SecurityHeaders/RateLimit/
> Logging), валидация Pydantic, fail-loud прод-проверка (`validate_production_security`), bandit +
> pip-audit в CI. Полный привязанный к коду срез — `docs/security.md`. Ниже — остаток до прод-уровня,
> разделённый на **бэкенд-код** (делаем здесь) и **деплой/орг** (веха деплоя — не дублируем, ссылка).

**Бэкенд-код (харденинг — здесь):**
- [ ] **Rate-limit на общий стор (Redis).** Сейчас счётчик in-memory на инстансе
  (`RateLimitMiddleware._hits`) — на мультиинстансном проде состояние не делится, лимит дырявый.
  Вынести в общий стор для горизонтального масштабирования. До масштабирования (один инстанс) —

## Сквозные правила (действуют на всех вехах)

1. **НИКОГДА не писать имена в гите / релизах / публичных артефактах.** Официально безымянно.
   Касается коммитов, тегов, GitHub Releases, публичного репо.
2. **4 аккаунта Claude: V, J, M, S.** Текущую вахту описывает аккаунт **J**. Имена — только для
   разграничения в разборах, на работу не влияют (WATCHLOG самодостаточен для любого).
3. **Вывод тестов — через `--junitxml`, не через `tail` по логу.** Итоговая строка pytest
   (`N passed`) и счётчики НЕ флашатся надёжно в перенаправленный файл (буферизация stdout, грабли
   песочницы). Решение: `pytest --junitxml=/tmp/result.xml`, парсить XML (точное число passed/failed),
   exit code как источник истины. Зафиксировано в `docs/pitfalls.md` и `docs/sandbox_runbook.md`.
4. **Тестирование мат-модели — СТРОГО вне проекта Claude** (Opus 4.8, без памяти/инструкций) —
   иначе эксперт biased. См. веху 7.
5. **TDD обязателен**, `app/core/` без согласования не трогать (исключение — `spending_advice.py`),
   схемные изменения проверять на PostgreSQL, без массового реформата кроме координированных окон,
   без эмодзи, русская проза + английский код.
