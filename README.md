# SPPR PersFin

Система поддержки принятия решений для персональных финансов на Python+FastAPI.

Проект умеет:
- вести журнал доходов и расходов;
- учитывать обязательные ежемесячные платежи;
- учитывать финансовые цели и накопления;
- рассчитывать ключевые показатели финансового состояния;
- формировать рекомендацию на основе текущих данных.

## Что используется в этом проекте

- Python 3.12
- FastAPI
- SQLAlchemy 2.0
- SQLite по умолчанию
- Jinja2
- HTML / CSS / JavaScript

## Структура проекта

Внутри проекта есть:

- `app/` — backend-приложение FastAPI
- `frontend/` — шаблоны и статические файлы интерфейса
- `data/` — каталог под данные, логи и подготовленные файлы
- `tests/` — каталог для автотестов
- `requirements.txt` — зависимости проекта
- `run.py` — локальный запуск через `uvicorn`

## Важное перед запуском

Проект можно запускать из любой папки на компьютере.

Главное условие:
- открыть терминал в корне проекта `sppr`, то есть в той папке, где находятся `run.py`, `requirements.txt` и каталог `app/`.

Все команды из инструкции ниже нужно выполнять именно из корня проекта `sppr`.

## Требования к окружению

Рекомендуемая версия Python:
- `Python 3.12`

Почему это важно:
- проект и зависимости проверялись на Python 3.12;
- на Python 3.13 некоторые зависимости могут устанавливаться нестабильно в зависимости от окружения.

Проверить установленный Python можно командой:

```
py --version
```

Если установлено несколько версий Python, для проекта лучше явно использовать Python 3.12.

## Создание виртуального окружения

Изначально папки `.venv` в проекте НЕТ.
Виртуальное окружение нужно создать локально на том компьютере, где проект будет запускаться.

Создавать `.venv` рекомендуется рядом с проектом, на одном уровне с папкой `sppr`.

Пример структуры:

```
persfin/
├── .venv/
└── sppr/
```

Если сейчас открыт терминал в папке `sppr`, то команда создания окружения будет такой:

```
py -3.12 -m venv ..\.venv
```

Если терминал открыт в родительской папке проекта, можно создать окружение так:

```
py -3.12 -m venv .venv
```

Принципиально важно только одно:
- после создания окружения путь к интерпретатору должен быть доступен;
- команды установки зависимостей и запуска нужно выполнять с использованием этого окружения.

## Активация виртуального окружения

Если `.venv` находится рядом с папкой `sppr`, то из каталога `sppr` активация выглядит так:

```
..\.venv\Scripts\Activate.ps1
```

Если PowerShell блокирует выполнение скриптов, можно использовать запуск без активации:

```
..\.venv\Scripts\python.exe -m pip install -r requirements.txt
..\.venv\Scripts\python.exe run.py
```

## Установка зависимостей

Из корня проекта `sppr` выполните:

```
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Если вы не активировали окружение, используйте полный путь к интерпретатору из `.venv`.

Пример:

```
..\.venv\Scripts\python.exe -m pip install --upgrade pip
..\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## Настройка переменных окружения

В проекте используется файл `.env`.

Перед первым запуском:
1. скопируйте файл `.env.example` в `.env`
2. при необходимости измените значения

В PowerShell это можно сделать так:

```
Copy-Item .env.example .env
```

Базовое содержимое `.env`:

```
DATABASE_URL=sqlite:///./sppr.db
DEBUG=true
API_PREFIX=/api
PROJECT_NAME=SPPR PersFin
```

Пояснение:
- `DATABASE_URL=sqlite:///./sppr.db` — локальная SQLite-база будет создана в корне проекта `sppr`
- `DEBUG=true` — удобно для локальной разработки
- `API_PREFIX=/api` — префикс API-маршрутов
- `PROJECT_NAME=SPPR PersFin` — название приложения

## Инициализация базы данных

Отдельно вручную инициализировать базу обычно не нужно.

При старте приложения вызывается `init_db()`, который:
- создаёт таблицы;
- для SQLite добирает недостающие колонки в текущей схеме, если это нужно.

То есть в обычном сценарии достаточно просто запустить приложение.

## Запуск проекта

Из корня проекта `sppr`:

```
python run.py
```

Если окружение не активировано:

```
..\.venv\Scripts\python.exe run.py
```

По умолчанию приложение настроено на:
- `host=127.0.0.1`
- `port=8000`

После запуска откройте в браузере:

```
http://127.0.0.1:8000
```

## Если порт 8000 занят

Если порт `8000` уже занят другой программой - приложение не запустится.

Самый простой вариант — запустить вручную на другом порту, например `8001`:

```
python -m uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
```

Или без активации окружения:

```
..\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
```

Тогда приложение будет доступно по адресу:

```
http://127.0.0.1:8001
```

## API

Основные маршруты:

- `GET /api/transactions` — список транзакций
- `POST /api/transactions` — создать транзакцию
- `DELETE /api/transactions/{transaction_id}` — удалить транзакцию
- `GET /api/obligations` — список обязательств
- `POST /api/obligations` — создать обязательство
- `DELETE /api/obligations/{obligation_id}` — удалить обязательство
- `GET /api/goals` — список целей
- `POST /api/goals` — создать цель
- `DELETE /api/goals/{goal_id}` — удалить цель
- `GET /api/analysis` — получить финансовые показатели
- `POST /api/recommendation` — получить рекомендацию
- `POST /api/demo/load` — загрузить демо-данные
- `POST /api/demo/clear` — очистить демо-данные

## Демо-режим

Для демонстрации системы можно использовать кнопки в интерфейсе:
- `Загрузить демо-данные`
- `Очистить данные`

Демо-набор включает:
- доходы;
- расходы;
- обязательство;
- финансовую цель.

Это удобно для показа проекта без ручного заполнения базы.

## Возможные проблемы

### 1. Не активируется PowerShell-скрипт окружения

Если возникает ошибка политики выполнения, используйте запуск без активации:

```
..\.venv\Scripts\python.exe run.py
```

### 2. Не запускается приложение из-за занятого порта

Запустите приложение на другом порту, например `8001`.

### 3. После обновления проекта база выглядит устаревшей

Если проект обновлялся поверх старой версии, можно:
- удалить файл `sppr.db`
- запустить приложение заново

После этого SQLite-база будет создана повторно.

## Передача проекта на другой компьютер

Перед передачей проекта на другой компьютер настоятельно рекомендуется:
- не включать папку `.venv` в архив или репозиторий;
- не включать `__pycache__`;
- при необходимости передавать файл `.env.example`, а не заполненный `.env`;
- передавать проект вместе с этой инструкцией.
