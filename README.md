# FINPILOT — СППР для персональных финансов

**FINPILOT** — система поддержки принятия решений (СППР), которая помогает грамотно
распределять свободные деньги между тремя направлениями: **досрочное погашение долга**,
**резерв ликвидности** и **финансовые цели**.

В основе — многокритериальная оптимизация (SAW), детерминированные финансовые показатели
и прогноз методом экспоненциального сглаживания с Monte-Carlo. Главный принцип —
**объяснимость**: каждая рекомендация сопровождается понятным «почему», без формул и жаргона.

---

## Возможности

- **Рекомендация распределения** свободного денежного потока с ранжированием альтернатив (SAW, 5 профилей риска).
- **Объяснимость** — человеческие объяснения вместо формул; алгоритм отказывается советовать при дефиците (fail-loud), а не фантазирует.
- **Прогноз** баланса и свободных денег на горизонт (SES + Monte-Carlo, доверительный интервал) с **алертом прогнозного дефицита**.
- **Категоризация транзакций** — детерминированный rules-engine (13 категорий, ключевые слова + MCC), без ML.
- **Импорт выписок** CSV (Тинькофф/Сбер/универсальный) с автокатегоризацией.
- **Категорийные бюджеты** (план-факт) и разрез расходов по категориям и мерчантам.
- **Сценарии «что если»** — пересчёт рекомендации под гипотетические доход/расходы с сохранением прогонов.
- **Светлая и тёмная темы**, адаптивный интерфейс.

## Архитектура

Слоистый монолит с однонаправленными зависимостями:

```
app/api        — HTTP-роуты (FastAPI), валидация запросов
   ↓
app/services   — оркестрация: pipeline, planning, forecasting, импорт выписок, логирование
   ↓
app/core       — чистая бизнес-логика: метрики, генерация/фильтрация/ранжирование
                 альтернатив, Avalanche, прогноз, категоризация
   ↓
app/database   — модели SQLAlchemy, CRUD, миграции Alembic
```

`app/core` не зависит ни от FastAPI, ни от БД — это делает ядро легко тестируемым (покрытие 88%).

## Быстрый старт через Docker

Поднимает приложение вместе с PostgreSQL одной командой:

```bash
docker compose up --build
```

Приложение будет доступно на `http://localhost:8000/dashboard`.

## Локальный запуск (macOS / Linux)

1. Создать и активировать виртуальное окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
2. Установить зависимости:
   ```bash
   pip install -r requirements.txt -r requirements-dev.txt
   ```
3. Применить миграции:
   ```bash
   alembic upgrade head
   ```
4. Запустить сервер:
   ```bash
   python3 -m uvicorn app.main:app --reload --port 8000
   ```
5. Открыть `http://127.0.0.1:8000/dashboard` (при обновлении статики — жёсткое обновление `Cmd/Ctrl+Shift+R`).

Для быстрого знакомства загрузите демо-данные: в боковой панели выберите один из шести
портретов пользователей — данные пересчитываются реальным движком.

## Переменные окружения

Скопируйте `.env.example` в `.env` и при необходимости отредактируйте:

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `ENVIRONMENT` | `development` | `development` \| `production` (влияет на reload, HSTS) |
| `DATABASE_URL` | `sqlite:///./finpilot.db` | строка подключения к БД |
| `CORS_ORIGINS` | `localhost:8000,...` | разрешённые источники через запятую |
| `RATE_LIMIT_REQUESTS` | `30` | лимит запросов на чувствительные эндпоинты за окно |
| `RATE_LIMIT_WINDOW_SECONDS` | `60` | размер окна rate-limit |

Секреты передаются только через `.env` и никогда не хранятся в коде.

## Качество и безопасность

- **Линт:** `ruff check app/ tests/`
- **Аудит зависимостей:** `pip-audit -r requirements.txt`
- **CI:** GitHub Actions прогоняет ruff → pytest → pip-audit → docker build на каждый push в `main`.
- **Безопасность:** OWASP-минимум (ORM, экранирование, валидация, CSRF, security-заголовки, rate-limit). Подробности — в [`SECURITY.md`](SECURITY.md).
