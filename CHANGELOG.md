## [3.5.0] — 2026-06-16 — Прогресс выплат, читаемость прогноза, ставки накоплений (MINOR)

### Добавлено
- **Прогресс выплат по обязательствам** — в карточке кредита, как у целей: «Выплачено X из Y · N% · осталось Z» + полоса прогресса прямо в основной строке (раньше прогресс был только в раскрытии). Без `start_date` показывается остаток долга.
- **Кнопка «Из моего вклада»** рядом с ползунком ставки накоплений — подставляет лучшую ставку среди вкладов пользователя (симметрично кнопке «Ставка ЦБ»).

### Изменено
- **Объяснение прогноза переписано для пользователя** — убраны академические обозначения (формулы, источники, «ВКР») из карточки графика. Вместо технической строки — понятное описание: что показывает прогноз, как он строится по шагам, почему ему можно доверять и как проверить самому. Backend-описание метода тоже очищено от внутренней нотации.
- **Подсказка к ставке накоплений** уточнена: ориентир привязан к ставкам вкладов, по умолчанию берётся лучшая.

## [3.4.0] — 2026-06-15 — UI-правки: разбор альтернатив, читаемость согласий, ставка ЦБ (MINOR)

### Добавлено
- **Разбор каждой альтернативы по клику** — в полном списке допустимых вариантов любая строка раскрывается: вклад каждого критерия в оценку (норм. значение × вес профиля), итоговый U и насколько вариант уступает рекомендованному (с указанием критерия, по которому он проседает).
- **Разбор отклонённых вариантов по клику** — для каждого отсеянного плана видно фактические Rt/Lt/Dt после распределения и явный список причин отклонения.
- **Индикатор активного тест-кейса** — загруженный портрет остаётся выбранным в списке и подсвечивается под селектором («Загружен кейс: …»); сохраняется между перезагрузками, сбрасывается кнопкой «Очистить».

### Исправлено
- **Текст согласий при регистрации не читался** — приглушённый серый → основной цвет, крупнее, чекбоксы с брендовым акцентом, блок вынесен в подложку.
- **Ставка ЦБ не подтягивалась** — `cbr_rate.py`: SOAPAction приведён к виду с кавычками (требование SOAP 1.1 / ASMX), добавлен реалистичный User-Agent (cbr.ru режет ботов), таймаут 4 → 6 с, реальная причина ошибки теперь пишется в лог вместо «молчания».
- **График прогноза был сжат** — блок прогноза вынесен из правой колонки на всю ширину страницы, viewBox расширен (760→1160) — линия и коридор больше не сдавлены.

### Изменено
- **Убраны эмодзи из интерфейса** — заголовок онбординга, кнопки экспорта/ставки/файла, плейсхолдеры и предупреждения переведены на чистые line-иконки и типографику (без цветных смайлов).

## [3.3.0] — 2026-06-15 — Продакшен-правки: UI, ставка ЦБ, накопления-буфер (MINOR)

### Добавлено
- **Ставка по накоплениям** — ползунок в планировании (0–30%) + кнопка «подставить ключевую ставку ЦБ». Сервис `cbr_rate.py` тянет ставку с cbr.ru с graceful fallback (`CBR_KEY_RATE_FALLBACK`), эндпоинт `/api/planning/key-rate`. Ставка передаётся как `r_bench` в расчёт и экспорт.
- **Учёт накоплений при отрицательном Rt** — вместо «денег нет» система считает буфер: на сколько месяцев накоплений (подушка + ликвидные) хватит покрывать дефицит, с предупреждением что это временно.
- **Детальный портрет кейса в валидации** — клик по кейсу разворачивает полный разбор (доходы, расходы, обязательства, цели с прогрессом, накопления, метрики). Эндпоинт `/api/demo/preview` (без записи в БД).
- **Кликабельная карточка обязательства** — раскрытие деталей (платёж, ставка, остаток, когда взят, сколько уже платите) + прогресс выплат. Демо-кейсы получают `start_date` для наглядности.

### Исправлено
- **Серые карточки** целей/обязательств в светлой теме → белые с тенью.
- **Отображение ставки обязательства** — `0.085%` → корректно `8.5%` (нормализация доли/процента).
- Все `alert()` заменены на единый тост (ошибки, удаление аккаунта, подтверждение email).
- **Фавикон** перегенерирован из центральной эмблемы логотипа (без артефактов «линий»).
- Селектор тест-кейсов получил плейсхолдер «— выберите тест-кейс —» (раньше всегда был выбран первый).

## [3.2.0] — 2026-06-15 — Согласия и онбординг (MINOR)

### Добавлено
- **Согласие на обработку ПДн (152-ФЗ)** при регистрации — обязательная галочка, без неё регистрация отклоняется (400). Фиксируется `users.consent_at`.

### Проверено
- Полный HTTP-смоук на запущенном `uvicorn` (не только TestClient): /docs, регистрация с согласием и без, расчёт плана, CSV-экспорт, профиль, онбординг.

## [3.1.1] — 2026-06-14 — Фиксы персистентности и подтверждения email (PATCH)

### Исправлено
- **Данные/аккаунты не сохранялись между деплоями**: имя тома Docker зависело от имени папки, поэтому развёртывание в новую папку создавало новый том. Зафиксировано имя `finpilot_pgdata` — данные переживают любой redeploy (кроме `down -v`).
- **Подтверждение email «ничего не приходит»**: в `ENVIRONMENT=production` без SMTP ссылка подтверждения скрывалась. Теперь при выключенном SMTP ссылка отдаётся всегда (self-hosted), показывается красивым тостом с кнопкой.

### Добавлено
- Обязательства: поле «когда взят» + прогресс «выплачено X из Y мес» с полоской (как у целей).
- Универсальный тост (`showToast`) вместо `alert()` для подтверждения email.
- Кнопка-пояснение графика прогноза сделана заметной.

## [3.1.0] — 2026-06-14 — Профиль, верификация почты, экспорт, UX (MINOR)

### Добавлено
- **Личный профиль** (`/profile`): данные аккаунта, смена имени и пароля, **удаление аккаунта** со всеми данными.
- **Подтверждение email**: письмо со ссылкой при регистрации, эндпоинты `/api/auth/verify` и `/api/auth/resend-verification`, статус в профиле. Без SMTP ссылка отдаётся в dev-режиме для локального теста.
- **Экспорт в CSV**: план распределения (`/api/planning/export.csv`) и журнал операций (`/api/transactions/export.csv`) — серверные эндпоинты со скачиванием файла.
- **Дашборд прогноза**: карточки с числами (баланс через N мес, свободные деньги, диапазон 80%, тренд).
- **Период трат на дашборде**: селектор окна (30/90/180/365 дней).
- **Печать рекомендации в PDF** через браузер.

### Исправлено
- **Рекомендации**: текст для альтернатив 2/3 (был пустой), дедупликация по фактическому распределению — топ-3 теперь реально разные и с понятным «куда пойдут деньги» в % и ₽.
- **Фильтр периода в журнале**: реакция на `input` (DuckDuckGo/WebKit не слал `change` до потери фокуса).
- **Генерация пароля браузером**: поля обёрнуты в `<form>`, `new-password` в регистрации.
- Раздел валидации: убраны упоминания ВКР.

### Изменено
- `users.email_verified` (миграция 0010).
- Прогноз отдаёт `stable_baseline` (регулярные доходы/расходы); математика SES/Monte-Carlo не изменена.

## [3.0.0] — 2026-06-12 — Международный / многопользовательский режим (MAJOR)

Ломающий релиз: схема БД расширена (`users`, FK `user_id`, денежные поля → `numeric`),
появилась аутентификация и изоляция данных по пользователю. Логика ядра СППР
(SAW + Avalanche + Monte-Carlo) не изменена — KEEP-01..05 сохранены.

### Added
- **Аутентификация (INFRA-06, NFR-05, DATA-03):** модель `User` (uuid), bcrypt-хеши,
  JWT (httpOnly-cookie + Bearer), маршруты `/api/auth/{register,login,logout,me}`.
- **Изоляция данных:** все пользовательские сущности получили FK `user_id`; роуты
  фильтруют по текущему пользователю. Анонимный режим (без токена) видит только
  данные без владельца.
- **Single→multi миграция без потери:** первый зарегистрированный пользователь
  усыновляет данные анонимного режима (`adopt_orphan_rows`).
- **Мультивалюта (FR-19, DATA-08):** поле `currency` на сущностях, `base_currency`
  у пользователя, таблица `fx_rates` (USD-пивот), `CurrencyConverter`, маршруты
  `/api/fx/{rates,convert}`.
- **Ingestion-слой (DATA-07, KEEP-07):** каноническая модель `FinancialSnapshot`,
  Protocol-контракты (`FinanceEngine`, `FinancialDataProvider`, `TokenStore`,
  `PlaidClient`), адаптер реального ядра `CoreFinanceEngine` (INFRA-15, REFACTOR-04).
- **Provider: Manual с персистентностью (INFRA-18):** `manual_snapshots`, репозиторий
  ручного ввода переживает рестарт.
- **Provider: Plaid (FR-18, INFRA-16/17):** `EncryptedTokenStore` (Fernet-шифрование
  токенов «в покое»), `PlaidProvider`, ленивый `RealPlaidClient`, маршруты
  `/api/plaid/{exchange,sync}`. Опционально — активируется `PLAID_*`.
- **B2B API (FR-23):** `/v1/analyze` принимает канонический снимок партнёра и
  возвращает рекомендацию; авторизация по `X-API-Key`. Активируется `B2B_API_KEYS`.
- **UI:** виджет входа/профиля в шапке, модалка вход/регистрация, `auth.js`
  (Bearer-перехват fetch).
- **Миграция 0009:** users, FK, numeric-типы, fx_rates, ingestion-таблицы; upgrade/downgrade.
- **Тесты:** `tests/test_auth.py` (изоляция, усыновление, хеши), `tests/test_international.py`
  (валюта, ingestion-движок, шифрование, B2B).

### Changed
- Денежные поля: `Float` → `Numeric(14,2)`, ставки → `Numeric(6,4)` во всех таблицах
  (точность денег, устранение ошибок округления).
- `UserPrefs`: одна строка на пользователя (`user_id` unique FK) + `base_currency`.
- `events`/`recommendations`: `user_id` `int` → `uuid`-строка.
- CSRF-middleware пропускает запросы по Bearer/API-Key (нет амбиентных cookie → нет CSRF-риска).
- Rate-limit распространён на `/api/auth/login`, `/api/auth/register`, `/v1/analyze`.
- `APP_VERSION` → `3.0.0`.

### Security
- Пароли — только bcrypt-хеш с солью (NFR-05).
- Plaid-токены шифруются Fernet «в покое» и не утекают в логи/URL (NFR-06, INFRA-17).
- Новые секреты (`JWT_SECRET`, `PLAID_*`, `TOKEN_ENCRYPTION_KEY`) — только через `.env`.

### Notes
- Совместимость: 105 существующих тестов проходят без изменений (анонимный режим = поведение v2.x).
- Open banking и B2B по умолчанию выключены (пустые ключи). На рынке РФ основной путь
  загрузки — импорт CSV-выписок (KEEP-06).
